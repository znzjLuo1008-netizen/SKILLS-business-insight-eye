## 这是什么？

这是一个 AI 驱动的财报分析工具（WorkBuddy Skill），能够：

1. **读取**上市公司财报 PDF（年报 / 季报）
2. **提取**全部关键财务数据（收益表 / 资产负债表 / 现金流量表）
3. **自动交叉验证**数据准确性（会计恒等式 / 杜邦拆解 / 业务线合计）
4. **生成**一份包含 20+ 可交互图表的单页 HTML 深度分析报告
5. **自检** 40+ 项 checklist 确保零错误

**严格基于 PDF 原文数据，禁止编造。** 每个数字都可以在财报中找到出处。

## 图表清单（20+）

### 可视化类型
- **Chart.js 图表**：饼图 / 柱状图 / 折线图 / 雷达图 / 水平条形图 / 堆叠图
- **CSS 桑基图**：资金流向（收入来源 → 业务板块 → 成本去向）
- **CSS Treemap**：资产负债表结构全景
- **CSS 漏斗图**：利润漏斗（营收 → 毛利 → 经营利润 → 净利 → FCF）
- **评分条**：风险因子评分 · 综合评定

### 交互特性
- **50+ 术语 hover 解释**：每个专业名词都有三层 tooltip（英文名 + 专业定义 + 大白话翻译）
- **移动端点击展开**：触屏设备点击术语即可展开解释
- **评分卡片 hover**：详细 tooltip 展示指标含义
- **响应式布局**：640px / 380px 双断点适配

## 技术栈

- **图表引擎**：Chart.js v4.4.7（CDN）
- **桑基图**：CSS Flex + SVG Path + JS 动态计算
- **Treemap**：纯 CSS Grid
- **漏斗图**：CSS clip-path
- **Tooltip**：CSS position:fixed + JS 动态定位
- **部署**：GitHub Pages（静态 HTML，零依赖）

```

Step 4: 生成 HTML 页面
  └─ 数据注入 → 图表渲染 → tooltip 生成

Step 5: 生成后自检（40+ 项 checklist）
  └─ 数据准确 ✅ 逻辑一致 ✅ 零残留 ✅ 移动端适配 ✅
```

## 12 条铁律（禁止编造数据）

| # | 规则 |
|---|------|
| 1 | **只用 PDF 原文数据** — 每个数字必须能在 PDF 中找到 |
| 2 | **不补历史数据** — PDF 只有 2 年就只展示 2 年 |
| 3 | **衍生指标公式算** — ROE/FCF 等必须计算，不估计 |
| 4 | **不确定就标注** — 口径不明写"需确认" |
| 5 | **宁可留空不编造** — 没数据就删图表 |
| 6 | **tooltip 贴合公司** — 不用其他公司比喻 |
| 7 | **文案基于 PDF** — Agent 判词用管理层讨论原文 |
| 8 | **市场声音默认删除** — 无真实来源不生成 |
| 9 | **终章无来源就删除** — 估值数据必须可溯源 |
| 10 | **DATA 禁止为空** — 防止 fallback 到默认值 |
| 11 | **图表适配期数** — 2 年数据不硬画 5 年折线 |
| 12 | **没数据就不放** — 空节点直接删除 |

## 设计原则

### 配色
- **整体基调**：白底 + 马卡龙淡色系
- **图表数据**：饱和度 40-55%，抓眼球
- **背景容器**：低饱和度 5-12%，呼吸感
- **涨跌色**：中国/港股 涨红跌绿 🔴🟢 · 美股 涨绿跌红 🟢🔴

### 排版
- 单页长滚动，无 Tab / 侧边栏
- 图表双列排列（移动端单列）
- 紧凑无留白，每个区域填满内容
- 业务线分组卡片（圆角 + 浅色背景 + 竖线连接子项）

### 移动端适配
- **640px 断点**：Score 4 列→2 列，图表双列→单列
- **380px 断点**：Score→1 列，canvas 高度缩小
- **触屏 tooltip**：点击展开/收起（替代 hover）
- **防溢出**：`overflow-x:hidden` + `max-width:100vw`

## How to Use This Skill

### 方式一：WorkBuddy 中使用
1. 将 `skill/` 文件夹安装为 WorkBuddy Skill
2. 上传财报 PDF
3. 触发 Skill → 自动生成分析页面

### 方式二：手动使用
1. Clone 本仓库
2. 用 `skill/references/template.html` 作为模板
3. 按 `skill/SKILL.md` 中的 5 步工作流操作
4. 填入数据到 `const DATA = {}` 对象
5. 按 `skill/references/validate_checklist.md` 自检

## 踩坑经验

通过 3 份报告的迭代，积累了 15 条实战经验：

- 模板替换法永远有残留 — 必须用数据驱动法
- `const DATA = {}` 空对象会 fallback 到默认值 — 必须填入真实数据
- 净现比不适用所有公司 — Tesla 用 FCF Margin 更合理
- 图表 Y 轴不调整 = 看不到波动 — 必须根据数据设置 min/max
- 桑基图空节点会显示空白色块 — 直接删除 HTML
- 估值温度计位置必须与 PE 一致 — 高估指针不能在"合理偏低"区
- 移动端 640px 断点被替换脚本误改为 52px — 需要验证

完整经验列表见 `skill/SKILL.md` 底部。
