# 小罗-商业透视眼🔍 v3.0

> 输入一家公司的财报 PDF，输出一份数据精准、可交互的专业分析网页。

## 🔴 核心原则

1. **一切数字必须来自 PDF 原文**——绝不编造、绝不套用历史模板数据
2. **提取先行，生成在后**——先完成全量数据提取和交叉验证，再生成页面
3. **图表为数据服务**——坐标轴、图表类型、维度名称全部跟随数据调整，不硬套模板
4. **tooltip 标题必须与字段语义一致**——品牌力→Brand Power，不是 Network Effect
5. **没有数据就不放**——桑基图空节点、无数据图表一律删除
6. **图表适配数据期数**——PDF 只有 2 年数据时：折线图→柱状对比图；5 年趋势图→2 年 YoY 对比
7. **所有 DATA 对象字段必须填入真实数据**——禁止 `const DATA = {}`（空对象会 fallback 到腾讯默认值）
8. **市场声音区域默认删除**——除非用户明确要求且提供了真实来源链接，否则不生成市场声音模块（禁止编造新闻/券商研报/名人评论）
9. **终章投资建议**——估值数据（PE对比/目标价/券商共识）如无真实来源一律删除，只保留基于 PDF 可计算的定量分析

## 工作流（5 步）

```
Step 1: PDF 全量数据提取（用 pdfplumber / PDF skill）
Step 2: 数据交叉验证（自动校验勾稽关系）
Step 3: AI 分析与评分（基于提取的真实数据）
Step 4: 生成 HTML 页面（复制模板 + 注入 DATA 对象）
Step 5: 生成后自检（40+ 项 checklist）
```

## ⛔ 最高优先级规则：禁止编造数据

> **PDF 里没有的数据，一律不用。宁可删图表也不编数据。**

1. **只用 PDF 原文数据**：每一个出现在页面上的数字，必须能在 PDF 中找到出处
2. **不从网上补历史数据**：如果 PDF 只有 2 年，图表改为 2 年对比，不画 5 年折线
3. **衍生指标必须公式算**：ROE = 归母净利/归母权益，不能"估计"或"参考行业"
4. **不确定就标注**：口径不明的数据在页面上标注"需确认"
5. **宁可留空也不编造**：没有数据的图表/卡片/Agent评语直接删除
6. **tooltip 内容必须贴合该公司**：不能用腾讯的比喻解释金山的业务
7. **所有文案（Agent 判词、上行/下行、策略建议）必须基于 PDF 中的真实业务描述**

### 数据来源优先级
1. PDF 原文数字 → **必须使用**
2. PDF 数字的衍生计算（标注公式）→ **可以使用**
3. PDF 管理层讨论的定性描述 → **可作为 Agent 评语参考**
4. 外部搜索 → **仅在用户明确要求时使用，标注来源**
5. AI 推测/估算 → **绝对禁止**

## Step 1: PDF 全量数据提取（最关键）

### 1.1 财务报表三张表提取

**收益表（Income Statement）**：

| 字段 | 提取方法 | 必须 |
|------|---------|------|
| 近5年/5期营收 | 收益表 "Revenue" / "营业收入" 行 | ✅ |
| 近5年/5期毛利 | 收益表 "Gross Profit" / "毛利" 行 | ✅ |
| 近5年/5期经营利润 | "Operating Profit" / "经营溢利" 行 | ✅ |
| 近5年/5期净利润 | "Net Income" / "本公司拥有人应占盈利" 行 | ✅ |
| 业务线拆分（3-5 条） | 分部资料 / Segment Information | ✅ |

**资产负债表（Balance Sheet）**：

| 字段 | 提取方法 | 必须 |
|------|---------|------|
| 总资产 | "Total Assets" / "资产总额" | ✅ |
| 总负债 | "Total Liabilities" / "负债总额" | ✅ |
| 股东权益 | "Total Equity" / "权益总额" | ✅ |
| 归母权益 | "Equity attributable to owners" | ✅ |
| 现金及等价物 | "Cash and Cash Equivalents" | ✅ |
| 有息负债 | "Borrowings" / "银行贷款" | ✅ |
| 主要资产科目（前5大） | 按金额排序 | ✅ |

**现金流量表（Cash Flow Statement）**：

| 字段 | 提取方法 | 必须 |
|------|---------|------|
| 经营活动现金流 | "Cash from Operating Activities" | ✅ |
| 资本支出 | "Purchase of PP&E" / "购买物业设备" | ✅ |
| 自由现金流 | = OCF - CapEx（手动计算） | ✅ |
| 分红总额 | "Dividends paid" | ⚠️ |
| 回购总额 | "Repurchase of shares" | ⚠️ |

### 1.2 衍生指标计算（必须自己算，不能猜）

```
毛利率 = 毛利 / 营收 × 100%
净利率 = 净利润 / 营收 × 100%
经营利润率 = 经营利润 / 营收 × 100%
ROE = 净利润 / 归母权益 × 100%（用年末值）
资产负债率 = 总负债 / 总资产 × 100%
净现比 = 经营活动现金流 / 净利润
FCF = 经营活动现金流 - 资本支出
FCF Margin = FCF / 营收 × 100%
杜邦拆解：净利率 × 资产周转率(=营收/总资产) × 权益乘数(=总资产/归母权益)
人均创收 = 营收 / 员工人数
```

### 1.3 提取注意事项

- **IFRS vs Non-IFRS / GAAP vs Non-GAAP**：标明使用的口径，全文保持一致
- **货币单位**：注意 PDF 中是百万/亿/千，统一转换
- **会计年度 vs 日历年度**：有些公司财年不是 1-12 月，需注意
- **重述数据**：如果 PDF 中历史年份数据有重述（Restated），用重述后的
- **一次性项目**：大额非经常性损益需标注（如 Tesla 2024 Q4 的 $5.9B 递延税）

## Step 2: 数据交叉验证（自校准）

**生成页面之前**，必须执行以下勾稽验证：

### 2.1 收益表勾稽
```python
assert abs(毛利率 - 毛利/营收*100) < 0.5  # 毛利率验证
assert abs(净利率 - 净利/营收*100) < 0.5  # 净利率验证
assert abs(sum(业务线) - 营收) / 营收 < 0.02  # 业务线合计 ≈ 总营收（允许2%舍入）
assert 毛利 > 经营利润 > 0 or 有合理解释  # 利润层级递减
```

### 2.2 资产负债表勾稽
```python
assert abs(总资产 - 总负债 - 股东权益) / 总资产 < 0.01  # 会计恒等式
assert abs(归母权益 + 少数股东权益 - 股东权益) / 股东权益 < 0.01
assert abs(资产负债率 - 总负债/总资产*100) < 0.5
```

### 2.3 现金流勾稽
```python
assert FCF == OCF - CapEx  # 自由现金流定义
assert abs(净现比 - OCF/净利润) < 0.05
```

### 2.4 跨表勾稽
```python
assert abs(ROE - 净利润/归母权益*100) < 1.0
# 杜邦拆解验证：净利率 × 周转率 × 杠杆 ≈ ROE
dupont = 净利率/100 * (营收/总资产) * (总资产/归母权益)
assert abs(dupont*100 - ROE) < 2.0
```

### 2.5 趋势合理性
```python
# 营收增长率 = (今年-去年)/去年
for i in range(1, 5):
    growth = (rev[i] - rev[i-1]) / rev[i-1] * 100
    # 增长率应在 -50% ~ 200% 之间（否则需特殊说明）
```

**如果校验不通过**：回到 Step 1 重新核对 PDF 原文，不能跳过。

## Step 3: AI 分析与评分

基于 Step 1-2 验证通过的真实数据，由 AI 生成以下分析内容：

### 3.1 八维度评分

| 维度 | 评分依据 | A级标准 | C级标准 |
|------|---------|---------|---------|
| 商业模式 | 业务线多元化、收入质量 | 3+引擎且最大≤50% | 单一业务>80% |
| 增长质量 | 营收增速+利润增速+毛利率趋势 | 双增速>15%+毛利率扩张 | 负增长+毛利率收缩 |
| 竞争力 | ROE+毛利率+人均创收 | ROE>15%+毛利率>40% | ROE<10%+毛利率<20% |
| 护城河 | 5维度综合 | 4维度≥4星 | 多维度≤2星 |
| 现金流 | FCF+FCF Margin+净现比 | FCF Margin>10%+净现比>1 | FCF为负 |
| 资产负债 | 负债率+净现金+流动比 | 负债率<40%+净现金 | 负债率>70% |
| 管理层 | 资本配置+股东回报+战略 | 持续回购分红+ROE提升 | SBC稀释严重+战略混乱 |
| 风险 | 6因子加权 | 综合<30分 | 综合>60分 |

### 3.2 颜色规则
- **A / A+ / A-** → `gd gd-a`（🟢 绿色）
- **B / B+ / B-** → `gd gd-b`（🟠 橙色）
- **C / C+ / C-** → `gd gd-c`（🔴 红色）

### 3.3 护城河 5 维度
- 根据**行业特征**选择维度名称（不固定为"网络效应/转换成本"等）
- 如：互联网公司 → 网络效应/转换成本/品牌/规模/数据壁垒
- 车企 → 品牌力/规模制造/充电网络/技术领先/成本优势
- 银行 → 牌照壁垒/资金成本/客户粘性/网点规模/品牌信任

### 3.4 风险因子
- 根据公司**实际面临的风险**选择 6 个因子（不固定为"地缘政治/AI回报"等）
- 每个因子独立评分 0-100，然后加权汇总
- 综合分 < 30 绿色，30-60 橙色，> 60 红色

### 3.5 图表个性化
- **净现比图**：适用于大多数公司；但如果净利润有大额一次性项导致净现比异常波动，替换为 **FCF Margin 趋势图**
- **桑基图**：节点数量和连线根据实际业务线数量调整，**没有数据的节点不放**
- **图表坐标轴**：
  ```
  y_min = floor(数据最小值 × 0.85)  # 留15%下方空间
  y_max = ceil(数据最大值 × 1.15)   # 留15%上方空间
  ```

### 3.6 Tooltip 规则
- 每个 tooltip 必须包含三层：`tp-pro`（英文专业名）+ `tp-def`（定义）+ `tp-ez`（大白话）
- **`tp-pro` 标题必须与所属字段的语义一致**
- 首次出现的专有名词都要加 tooltip（GAAP/IFRS/FSD/EPS/SBC/COGS 等）
- 大白话中的比喻要**贴合该公司业务**（不能用"聊天软件"比喻车企）

## DIM1 业务线分组样式规则

业务线拆分使用**分组卡片**样式（而非平铺列表）：
- 每个主业务板块用独立卡片（圆角12px + 浅色背景 + 同色边框）
- 子项通过 `margin-left:16px` 缩进 + 左侧竖线连接
- 不同板块用不同色系（紫色/绿色/粉色/蓝色）
- 卡片之间 8px 间距

```html
<!-- 示例结构 -->
```

## 踩坑经验

- 模板替换法永远有残留——必须用数据驱动法（先提取 JSON，再注入页面）
- tooltip 的 tp-pro 标题最容易错配——护城河5维度的字段名和英文标题必须对应
- 净现比不适用于所有公司——有大额一次性项的公司用 FCF Margin 更合理
- 图表 Y 轴不调整 = 看不到波动 = 图表无意义——必须根据数据范围设置 min/max
- 估值温度计位置必须与 PE 倍数一致——高估的股票指针不能在"合理偏低"区
- 风险评分颜色要与分值一致——52/100 中等风险不能用绿色底
- Treemap 数据必须来自资产负债表原文——不能套用其他公司的科目结构
- 桑基图空节点（flex:0）会显示为空白色块——没有数据的节点直接删除
- 杜邦拆解的三个因子相乘必须 ≈ ROE——如果对不上说明某个因子算错了
