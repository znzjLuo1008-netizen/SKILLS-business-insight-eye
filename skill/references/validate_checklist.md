# 商业透视眼 v3.0 — 生成后自检清单

> 每次生成新报告后，逐项执行以下检查。全部通过才能交付。

## 1️⃣ 数据准确性（最高优先级）

### 收益表
- [ ] 5年营收数据与 PDF 一致
- [ ] 5年净利润数据与 PDF 一致（标注 IFRS/Non-IFRS 口径）
- [ ] 业务线拆分合计 ≈ 总营收（允许2%舍入）
- [ ] 毛利率 = 毛利/营收×100%（重新计算验证，允许0.5pp偏差）
- [ ] 净利率 = 净利/营收×100%
- [ ] 经营利润率 = 经营利润/营收×100%

### 资产负债表
- [ ] 总资产 = 总负债 + 股东权益（允许1%舍入）
- [ ] 归母权益 + 少数股东权益 = 股东权益
- [ ] 资产负债率 = 总负债/总资产×100%
- [ ] Treemap 所有科目金额来自 PDF 原文
- [ ] 净现金 = 现金及等价物 - 有息负债

### 现金流
- [ ] FCF = 经营活动现金流 - 资本支出
- [ ] 净现比 = OCF / 净利润（标注口径）
- [ ] FCF Margin = FCF / 营收×100%

### 衍生指标
- [ ] ROE = 净利润 / 归母权益×100%
- [ ] 杜邦拆解：净利率 × 周转率 × 杠杆 ≈ ROE（允许2%偏差）
- [ ] 人均创收 = 营收 / 员工人数

### 验证脚本
```python
# 在生成完成后运行
html = open('输出文件.html').read()
import re
charts = re.findall(r"data:\[([\d.,\s-]+)\]", html)
print("所有图表数据数组:")
for i, d in enumerate(charts):
    print(f"  [{i}] [{d.strip()}]")
# 手动与 PDF 原文逐项对比
```

## 2️⃣ 逻辑一致性

- [ ] 估值温度计位置与 PE 倍数一致
  - PE > 行业均值 2x → 当前指针在"偏高"区（70-85%）
  - PE < 行业均值 → 当前指针在"合理偏低"区（20-35%）
- [ ] 巴菲特清单判定与数据一致
  - ROE < 15% → ⚠️ 而非 ✅
  - PE > 40x → ❌ 估值偏高
  - 资产负债率 > 60% → ⚠️
- [ ] 风险总分颜色正确
  - < 30 → 绿色底（--gnl）
  - 30-60 → 橙色底（--aml）
  - \> 60 → 红色底（--rdl）
- [ ] 风险总分大数字 = 文字中的分数
- [ ] 评分等级颜色正确（A→gd-a绿，B→gd-b橙，C→gd-c红）
- [ ] 大师判词中的数字与图表数据一致
- [ ] Agent 评语中的数字与图表数据一致

## 3️⃣ Tooltip 一致性

- [ ] 护城河5维度：tooltip tp-pro 标题与字段名语义一致
  - ❌ 品牌力 → Network Effect
  - ✅ 品牌力 → Brand Power
- [ ] 风险6因子：tooltip tp-pro 标题与因子名一致
  - ❌ 竞争加剧 → Regulatory Risk
  - ✅ 竞争加剧 → Competitive Intensity
- [ ] hover 标签数量 = 条形图因子数量
- [ ] 所有专有名词有 tooltip（GAAP/EPS/FCF/ROE/FSD 等）
- [ ] 大白话比喻贴合该公司业务（不用聊天软件比喻车企）

## 4️⃣ 图表质量

- [ ] 所有图表 Y 轴范围合理（数据波动清晰可见）
- [ ] 桑基图无空节点（没有 flex:0 的白色色块）
- [ ] 桑基图连线不引用已删除的节点
- [ ] 利润漏斗层级递减（营收 > 毛利 > 经营利润 通常 > 净利）
- [ ] 涨跌色符合市场惯例（中国涨红跌绿/美国涨绿跌红）
